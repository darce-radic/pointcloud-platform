# PRD-01: Data Ingestion, Harmonization & Storage

**Module:** Core Infrastructure
**Status:** Draft
**Target Audience:** Claude Code

## 1. Overview
This module handles the secure upload of raw point cloud files, standardizes them through an automated harmonization pipeline (noise reduction, GDPR blurring), and converts them into the Cloud Optimized Point Cloud (COPC) format for web streaming.

## 2. User Stories
- As a user, I want to upload massive (10GB+) LAS/LAZ files directly from my browser without the upload timing out.
- As a user, I want the system to automatically remove noise and classify the ground so my point cloud looks clean.
- As a compliance officer, I need all faces and license plates in the 360° imagery to be blurred automatically (GDPR compliance).
- As a user, I want my uploaded files to be instantly streamable in the 3D viewer without downloading them first.

## 3. Architecture & Storage Strategy
- **Primary Storage:** Cloudflare R2 (S3-compatible, zero egress fees).
- **Format:** COPC (Cloud Optimized Point Cloud) for 3D data.
- **Upload Pattern:** Presigned S3 URLs generated by the API, allowing the frontend to `PUT` directly to R2.
- **Processing Queue:** Supabase `processing_jobs` table.

## 4. Technical Specifications

### 4.1. Database Schema Additions
Ensure the `datasets` table in Supabase has the following columns (currently missing in migration 001):
```sql
ALTER TABLE public.datasets 
ADD COLUMN IF NOT EXISTS copc_url TEXT,
ADD COLUMN IF NOT EXISTS dtm_url TEXT;
```

### 4.2. API Endpoints (`api/routers/datasets.py`)
**`POST /projects/{id}/datasets/upload-url`**
- **Action:** Generates a presigned R2 URL for `PUT` operations.
- **Returns:** `{ "upload_url": "...", "dataset_id": "uuid", "r2_key": "raw/uuid.laz" }`

**`POST /datasets/{id}/complete-upload`**
- **Action:** Updates dataset status to `uploaded`.
- **Action:** Inserts a row into `processing_jobs` with `job_type = 'harmonization'`.

### 4.3. Harmonization Worker (`workers/harmonization/src/worker.py`)
*Note: This is a new worker to be created, replacing the direct-to-tiling flow.*

**Trigger:** Polls `processing_jobs` for `job_type='harmonization'` and `status='queued'`.
**Pipeline:**
1. Download raw file from R2.
2. Run PDAL pipeline:
   - `filters.outlier` (Statistical Outlier Removal for noise).
   - `filters.smrf` (Simple Morphological Filter for ground classification).
   - `filters.voxelcentroidnearestneighbor` (Density normalization).
3. Run GDPR blurring (YOLOv8) on associated panoramic imagery (if present).
4. Output harmonized `.laz` file.
5. Update `processing_jobs` status to `completed` and enqueue a `tiling` job.

### 4.4. Tiling Worker (`workers/tiling/src/worker.py`)
**Trigger:** Polls `processing_jobs` for `job_type='tiling'` and `status='queued'`.
**Pipeline:**
1. Download harmonized `.laz` file.
2. Run `pdal translate in.laz out.copc.laz writers.copc`.
3. Upload `out.copc.laz` to R2.
4. Update `datasets.copc_url` in Supabase.
5. Update `processing_jobs` status to `completed`.

## 5. Acceptance Criteria
- [ ] Files > 5GB can be uploaded successfully via the frontend using the presigned URL.
- [ ] The `datasets` table correctly reflects the upload status (`pending` -> `uploaded` -> `processing` -> `ready`).
- [ ] The harmonization worker successfully removes statistical outliers from a noisy test dataset.
- [ ] The tiling worker outputs a valid COPC file that can be read by `pdal info`.
- [ ] Both workers update their job progress in the `processing_jobs` table, which is reflected in the frontend UI.
