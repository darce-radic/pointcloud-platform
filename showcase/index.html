How the Agent Works — 5 Stages

Every natural language request passes through a deterministic LangGraph pipeline that guarantees valid, deployable output.

01
Intent Router
Classifies the message as a workflow request, data query, or general chat. Routes to the correct downstream handler without ambiguity.
02
Workflow Planner
Breaks the request into 3–7 ordered processing steps using a structured chain-of-thought prompt. Each step is named, typed, and given a clear purpose.
03
Node Selector
Uses pgvector semantic search across the 30+ node library to find the exact right n8n node for each planned step. Retrieval-augmented, not hallucinated.
04
JSON Generator
Constructs the complete n8n workflow JSON with Pydantic-enforced schema validation. Invalid workflows are automatically corrected before output.
05
Deployer & Publisher
Pushes the workflow to the n8n REST API and offers to add it to the viewer toolbar as a one-click tool for non-technical users.

Tools Appear Automatically

The viewer toolbar is driven entirely by the workflow_tools database table. When the AI agent publishes a new workflow, it appears in the toolbar immediately — no frontend code changes required.

workflow_tools — Supabase SQL 
-- Query published tools for an org
SELECT id, name, icon,
       description,
       required_inputs,
       n8n_workflow_id
FROM   workflow_tools
WHERE  organization_id = 'org_xyz'
  AND  is_published = true
ORDER BY sort_order;
